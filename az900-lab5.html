    Lab Overview

    Governance is the set of mechanisms that ensure Azure resources are deployed, configured, and used in accordance with organisational policy. Without governance, teams can deploy unsecured resources, incur unexpected costs, or violate compliance requirements. This lab covers the four main governance tools in Azure.

    ToolWhat it doesExam relevance
    Azure PolicyEnforces rules - e.g. all VMs must use approved sizes, all resources must have tagsAZ-900 + AZ-500
    Resource LocksPrevents accidental deletion or modification of critical resourcesAZ-900 + AZ-500
    Cost Management + BudgetsMonitors spend and triggers alerts when thresholds are reachedAZ-900
    Azure AdvisorPersonalised recommendations across reliability, security, performance, costAZ-900
    i All features in this lab are free. Azure Policy, resource locks, and Cost Management are included with every Azure subscription at no additional cost.
    Your Scenario

    Contoso Ltd's finance team has flagged unexpected Azure spend last month. Your manager wants you to implement governance controls: enforce a tagging policy so every resource is tagged with a cost centre, lock the production resource group to prevent accidental deletion, and set up a budget alert at 80% of the monthly limit. You also need to review Azure Advisor to see if any quick wins exist to reduce cost and improve security.

    Part 1 - Azure Policy

    Azure Policy evaluates resources against defined rules and can either audit (report non-compliance) or enforce (deny the deployment). Policies can be applied at management group, subscription, or resource group scope and inherit downward.

    TASK 1.1Explore the Policy service
    1. In the Azure portal, search for Policy and click the result
    2. On the Overview page, review the compliance summary - with a free account and minimal resources you will likely see high compliance
    3. Click Definitions in the left menu - browse the built-in policy library
    4. In the search box, type tag - review the tag-related built-in policies
    5. Click on Require a tag on resources to read the policy definition - note the JSON structure
    6. Click Initiatives - note the difference between a single policy and an initiative (group of policies)
    REFERENCE - Azure Policy Overview
    Azure Policy overview - the compliance dashboard shows how many resources comply with assigned policies. Source: Microsoft Learn
    i Policy vs Initiative: A policy is a single rule. An initiative is a collection of related policies assigned together - for example, the ISO 27001 initiative contains dozens of individual policies covering security controls.
    TASK 1.2Assign a tagging policy to your resource group
    1. In Azure Policy, click Assignments in the left menu then + Assign policy
    2. Under Scope, click the three dots and select your subscription then rg-contoso-labs
    3. Under Policy definition, click the three dots and search for Require a tag on resources - select it
    4. Click Next to reach Parameters - set the tag name to CostCentre
    5. Click Next then Review + create then Create
    6. Return to Policy - Assignments and confirm the new assignment appears
    ! Policy evaluation is not immediate - it can take up to 30 minutes for new assignments to evaluate existing resources. For this lab you are configuring the assignment and verifying it appears correctly, not waiting for the full evaluation cycle.
    + This policy will now audit new resources created in rg-contoso-labs to check they have a CostCentre tag. The effect is Audit by default - it reports but does not block. To enforce (deny untagged resources) the effect would be set to Deny.
    TASK 1.3Explore compliance and remediation
    1. Navigate to Policy - Compliance
    2. Find your tagging policy assignment - note the compliance state
    3. Click into the policy to see which resources are non-compliant
    4. Click Remediation in the left menu - explore how remediation tasks work for policies that support it
    i AZ-500 link: In AZ-500 you will use policy initiatives like Azure Security Benchmark and regulatory compliance frameworks (ISO 27001, SOC 2, PCI DSS). These work exactly the same way - same interface, larger scope.

    Part 2 - Resource Locks

    Resource locks prevent users from accidentally deleting or modifying critical resources - even if they have Owner-level RBAC permissions. Locks override RBAC for the specific operations they restrict.

    Lock typeCan read?Can modify?Can delete?Use case
    CanNotDeleteYesYesNoProduction resources that might need config changes but must not be deleted
    ReadOnlyYesNoNoCritical config resources - treat as read-only infrastructure
    ! Locks override RBAC. An Owner cannot delete a locked resource unless they first remove the lock. This is intentional - it protects against both accidental and malicious deletion.
    TASK 2.1Apply a CanNotDelete lock to your resource group
    1. Navigate to rg-contoso-labs
    2. In the left menu, click Locks
    3. Click + Add
    4. Set Lock name to rg-contoso-labs-lock
    5. Set Lock type to CanNotDelete
    6. Add a note: Production lab resource group - do not delete
    7. Click OK
    REFERENCE - Resource Locks Blade
    The Add lock form - set Lock type to CanNotDelete to prevent deletion while still allowing modifications. Source: Microsoft Learn
    TASK 2.2Test the lock - attempt to delete the resource group
    1. From the rg-contoso-labs overview, click Delete resource group in the command bar
    2. Type the resource group name to confirm and click Delete
    3. Azure will return an error - the lock is preventing deletion
    4. Return to Locks and verify the lock is still in place - do NOT remove it yet
    + The error you saw confirms the lock is working correctly. Even with Owner permissions, the resource group cannot be deleted while the lock exists. This is a key governance control for production environments.

    Part 3 - Cost Management & Budgets

    Azure Cost Management lets you monitor spend, analyse cost trends, and set budgets with automated alerts. It is free to use and works across all subscriptions.

    Example Budget Visualisation
    Monthly budget: $50 | Spend to date: $12 | Alert at: 80%
    $024% usedAlert at 80%$50
    TASK 3.1Explore Cost Analysis
    1. Search for Cost Management in the portal and click the result
    2. Click Cost analysis in the left menu
    3. Review the default view - it shows your subscription spend over the current billing period
    4. Use the Group by dropdown to switch between grouping by Service, Resource group, and Tag
    5. Change the date range to Last 3 months to see spending trends
    6. Click View to explore saved views - try the Cost by resource view
    i The Group by Tag view is why tags matter for cost management. If you applied the CostCentre tag in Lab 1, you can now filter costs by cost centre here. This is how organisations do chargeback to departments.
    TASK 3.2Create a cost budget with alert
    1. In Cost Management, click Budgets then + Add
    2. Set Name to AZ900-Lab-Budget
    3. Set Reset period to Monthly
    4. Set Amount to 50 (this is your monthly budget in dollars)
    5. Click Next: Alerts
    6. Click + Add alert condition - set type to Actual, threshold to 80 percent
    7. Enter your email address in the Alert recipients field
    8. Click Create
    + You will now receive an email alert if your subscription spend reaches 80% of your $50 budget in any calendar month. This is the minimum governance control every Azure user should have active.
    TASK 3.3Use the Azure Pricing Calculator
    1. Open a new browser tab and navigate to azure.microsoft.com/pricing/calculator
    2. Click Virtual Machines to add a VM to the estimate
    3. Set region to UK South, OS to Windows, tier to Basic, size to B1s
    4. Note the monthly cost estimate - this is the same VM type used in Lab 2
    5. Add a Storage Account to the estimate and configure it with LRS redundancy
    6. Review the total monthly estimate at the bottom of the page
    7. Click Export to download the estimate as an Excel file for your records
    i The Pricing Calculator is a public tool - no Azure account required. Use it before any deployment to estimate costs. The Total Cost of Ownership (TCO) Calculator (separate tool) compares on-premises vs Azure costs.

    Part 4 - Azure Advisor

    Azure Advisor analyses your subscription configuration and usage, then provides personalised recommendations across five categories: Cost, Security, Reliability, Operational Excellence, and Performance. It is free and requires no configuration.

    TASK 4.1Review Advisor recommendations
    1. Search for Advisor in the portal and click the result
    2. Review the Overview - note your score across all five categories
    3. Click Security - review any security recommendations for your subscription
    4. Click Cost - note any cost optimisation opportunities (may be limited with a free account)
    5. Click Reliability - review high availability recommendations
    6. For any recommendation, click through to read the detail - note the estimated impact and how to remediate
    REFERENCE - Azure Advisor Overview
    Azure Advisor dashboard - five categories with scores and recommendations. Security recommendations link directly to Defender for Cloud. Source: Microsoft Learn
    i Advisor and Defender for Cloud: Security recommendations in Advisor come from Microsoft Defender for Cloud. You will explore Defender for Cloud directly in Lab 6. Think of Advisor as the summary view and Defender for Cloud as the detailed security posture manager.

    Knowledge Check - AZ-900 Exam Style
    Click an answer to check it. One attempt per question. 0 / 5
    1. A company wants to prevent anyone from deleting a critical storage account, even administrators. Which Azure feature should they use?
    • A. Azure Policy with Deny effect
    • B. RBAC Reader role on the storage account
    • C. Resource lock with CanNotDelete type
    • D. Resource lock with ReadOnly type
    i CanNotDelete locks prevent deletion while still allowing reads and modifications. ReadOnly prevents all changes including modifications. Azure Policy controls what can be deployed but does not protect existing resources from deletion.
    2. Which Azure Policy effect should be used to prevent resources being created without a required tag?
    • A. Audit
    • B. Deny
    • C. Append
    • D. Disabled
    i Deny blocks non-compliant resource creation. Audit only logs and reports non-compliance without blocking. Append adds properties to resources. Disabled turns the policy off.
    3. Which tool would you use to estimate the monthly cost of a new Azure deployment before building it?
    • A. Azure Pricing Calculator
    • B. Azure Cost Management
    • C. Azure Advisor
    • D. Azure Monitor
    i The Pricing Calculator estimates future costs for planned deployments. Cost Management tracks actual spend on existing resources. Advisor gives recommendations on deployed resources. Monitor tracks performance and metrics.
    4. Azure Advisor provides recommendations across five categories. Which of these is NOT one of them?
    • A. Cost
    • B. Security
    • C. Reliability
    • D. Compliance
    i Azure Advisor's five categories are: Cost, Security, Reliability, Operational Excellence, and Performance. Compliance is managed through Azure Policy and Defender for Cloud, not Advisor directly.
    5. A ReadOnly lock is applied to a resource group. What happens when a user with Owner RBAC permissions tries to add a new resource to that group?
    • A. The resource is created - Owner permissions override locks
    • B. The operation is blocked - locks override RBAC permissions
    • C. The resource is created with a warning logged to Activity Log
    • D. The user must request lock removal approval before proceeding
    i Resource locks override RBAC permissions for the operations they restrict. A ReadOnly lock prevents all write operations including adding new resources, regardless of the user's role. The lock must be removed first.
